# 流程级突变探针(flow probe)· 设计

日期:2026-07-16
背景:纵向三部曲(trace #19 → 分相 #20 → 单例切窗 #21)让流程"可看";本项让流程"可考"——verify 哲学(理解靠撞真实运行验证)的纵向版:斩链上一步、预测流程断不断、真跑单 example 揭晓。

## 1. 已定决策(用户逐项确认)

- 考试形态:**读者自选步**(否掉考官模式/两者兼有——出题启发式留二期)。
- 刀落点:**方案丙**——命中方法内为主,失败回退文件级并显式标注(否掉纯甲:部分步不可探;否掉纯乙:假阴性腐蚀考试可信度)。

## 2. 目标与 CLI

`flow probe <flowId> --step <N> --predict red|green [--repo <p>] [--out <d>]`:斩流程第 N 步、真跑该流程的单 example、比对预测、产出 `easyreview.flowprobe.md`。**不落盘 flows.json**(探针是一次性考试,报告即产物)。

## 3. 前提与拒绝

- 只接受 `source.kind === 'rspec-trace'` 且 **source.spec 带行号**(单例流程)——全谱流程的"断不断"被其它 example 污染,友好拒绝并提示「用 spec:行号 重新 trace 单例流程」。
- flowId 不存在、--step 越界(非 1..steps.length)、--predict 非 red/green:各自友好拒绝。

## 4. 刀落点规则(方案丙)

- **主路径**:该步的 `methods` ∩ rawTrace 中该文件的方法定义行 → tree-sitter 定位包含该定义行的方法体 → 在**方法体范围内**用既有位点逻辑(verify/pick-site 的 Ruby 语句偏好)挑可注释语句。斩的保证是流程真正执行过的方法。
- **回退**:方法体内找不到好语句 → 既有文件级 chooseMutation;报告**显式标注**「刀落在流程未必经过的位置——绿色结果不可作为理解凭据」。
- 全文件都找不到位点 → 友好拒绝(与 verify 的 uncovered 先例一致)。

## 5. 执行与判定

沙箱同步 → `withMutation` 斩(finally 字节还原,真实仓零接触)→ runner cmd 跑 `spec:行号` → `parseRspecJson` → **red = 该 example 失败或加载崩**(加载崩=正常爆炸半径,rspec 探针先例)→ 与预测比对。绿(斩了没断)如实呈现两种解释:防御性代码 or 该行不在断言路径上。

## 6. 报告(easyreview.flowprobe.md,镜像 verify.md 风格)

流程名 / 步号与步文件 / 刀落点(文件:行 + 方法名 + 主路径|回退标注)/ 你的预测 / 实际结果 / 判定:✅ 预测命中——你懂这一步在流程里的角色;❌ 未命中——回去重读该步。

## 7. 测试计划(预计 +11 条)

- 方法体范围定位纯函数:含定义行的 method 节点、多方法挑对、找不到给回退信号。
- 判定纯函数:红绿 × 预测四象限。
- CLI 编排(fake exec):命中路径、未命中路径、回退标注出现在报告、报告文件落盘。
- 拒绝路径:全谱流程 / flowId 不存在 / step 越界 / predict 非法。

## 8. 真仓验收

- 「发消息·单例」流程:斩 request 段 message_builder 步 → 预测 red → 真跑变红 → 判定通过(核心物证)。
- 挑一个防御性/旁路步呈现诚实绿(报告的两种解释文案)。
- 构造一次回退标注场景。
- 真实仓零接触、沙箱字节还原、探针后 flows.json 未被改动;umwelt 零接触。

## 9. 不做什么

- UI 集成(流程 Tab 探针按钮)——二期。
- 考官模式(系统出题启发式)。
- 多步批量、探针历史/成绩单。
- 非 Ruby 栈(流程数据目前只有 rspec 来源)。
